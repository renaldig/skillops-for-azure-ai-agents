# Example SkillOps Inventory

| Task | Who asks for it | Current source of guidance | Risk if done inconsistently | Skill candidate |
|---|---|---|---|---|
| Review an App Service deployment before staging | Developer | Pull request comments and senior engineer | Missing identity, monitoring, or rollback decisions | Yes |
| Investigate API latency with Application Insights | On-call engineer | Wiki and saved KQL | Slow diagnosis and unsupported root-cause claims | Yes |
| Choose a naming prefix for a one-off demo | Developer | Team convention | Minor inconsistency | Maybe |
| Approve a production emergency change | Incident commander | Change process and human judgment | Unsafe automation of accountable decisions | No |
