# Azure Skill Design Review Checklist

- [ ] The skill targets a specific task.
- [ ] The description clearly states when to use the skill.
- [ ] Required inputs and missing-context questions are visible.
- [ ] The workflow is ordered and reviewable.
- [ ] Decision rules and constraints are explicit.
- [ ] Production, identity, secrets, data, cost, and networking guardrails are explicit.
- [ ] Output separates evidence, risk, questions, and next steps.
- [ ] Volatile pricing, quota, region, and preview facts require current verification.
- [ ] Core instructions are portable across supported hosts.
- [ ] Tools, network needs, permissions, credentials, and role assignments stay outside the core workflow.
