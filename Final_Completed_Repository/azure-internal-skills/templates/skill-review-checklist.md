Skill change checklist
[ ] The skill description clearly says when to use the skill.
[ ] The skill does not claim to approve work on behalf of a team.
[ ] Required internal standards are referenced or summarized correctly.
[ ] The skill says what information the assistant must not invent.
[ ] The skill separates required fixes from recommendations.
[ ] Human review boundaries are visible.
[ ] Examples use safe placeholder values, not real secrets or data.
[ ] The catalog owner and maturity fields are updated if needed.
[ ] At least one realistic test prompt was tried before approval.
