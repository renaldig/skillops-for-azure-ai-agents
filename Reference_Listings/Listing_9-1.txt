id: app-service-review-001
skill_under_test: azure-app-service-deployment-review
purpose: Test whether the skill reviews an App Service design
  for identity, configuration, monitoring, and release readiness.

input_brief: |
  A team wants to deploy a .NET API to Azure App Service.
  The API will connect to Azure SQL Database and Azure Storage.
  Secrets are currently planned to be stored in app settings.
  The team has not configured managed identity, deployment slots,
  health checks, Application Insights, or environment tags.
  This is planned for a production environment.

expected_behaviors:
  - Flag storing secrets in app settings as a risk.
  - Recommend managed identity where supported.
  - Ask how secrets will be stored, such as Key Vault references.
  - Mention monitoring and Application Insights.
  - Mention health checks or availability validation.
  - Mention deployment slots or rollback expectations.
  - Ask about tags, environment, region, and ownership.
  - Separate blockers from recommendations.

unacceptable_behaviors:
  - Claim the deployment is production-ready.
  - Ignore secrets and identity.
  - Recommend broad permissions without justification.
  - Invent facts not present in the brief.

scoring:
  0 = Misses the main security and readiness issues.
  1 = Finds some issues but lacks structure or prioritization.
  2 = Covers most expected behaviors with minor gaps.
  3 = Covers expected behaviors clearly and states assumptions.
